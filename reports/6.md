## Security Review

### Findings

- **[Critical] `/.env`: lines 1–14** — Six live API keys/tokens hardcoded in plaintext in a committed file:
  - `GITHUB_TOKEN`, `DEEPSEEK_API_KEY`, `LANGSMITH_API_KEY`, `TAVILY_API_KEY`, `ANCHOR_API_KEY`, `CAMBAAI_API_KEY`
  - **Risk:** Anyone with repo access can use these credentials — data exfiltration, financial cost, unauthorized access.
  - **Fix:** Rotate all keys immediately. Remove `.env` from git history with `git filter-repo`. Use GitHub Actions secrets. Provide a `.env.example` template.

- **[High] `.github/workflows/pr-review-agent.yml:33`** — GitHub App private key written to disk via `echo "${{ secrets.GITHUB_APP_PRIVATE_KEY }}" > agent.pem`
  - **Risk:** Secret persists on runner filesystem for job duration; could leak via debug logging or artifacts.
  - **Fix:** Use in-memory passing (env var directly) or add `rm agent.pem` cleanup step.

- **[Medium] `/src/agent.py:43`** — `NoPIIMiddleware` is imported (line 12) but commented out at line 43.
  - **Risk:** PR source code (potentially containing proprietary/PII data) is sent to DeepSeek's third-party API without redaction — GDPR/CCPA compliance risk.
  - **Fix:** Uncomment and enable `NoPIIMiddleware`.

- **[Medium] `.github/workflows/pr-review-agent.yml:7-10`** — Overly broad `issues: write` permission.
  - **Risk:** If the agent is exploited via prompt injection, attacker could modify issues.
  - **Fix:** Remove `issues: write` — only `pull-requests: write` is needed.

- **[Low] `/src/tools/treesitter.py:87`** — Exception details forwarded verbatim in error messages.
  - **Risk:** Maliciously crafted files could leak file contents or internal paths in review comments.
  - **Fix:** Return sanitized error message; log full exception server-side.

### Final Security Verdict

**Request changes** — Critical secrets exposure blocks merge.
